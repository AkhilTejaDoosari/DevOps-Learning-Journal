<summary><strong>4. Inside the Route 53 Engine (Solution)</strong></summary>

---

Route 53 runs a globally distributed network of DNS servers (AWS edge locations).
Every query is answered by the **nearest healthy server**.

### How It Works

1. User enters the domain (`chillspot.com`).
2. The request hits the nearest Route 53 edge server.
3. That server looks inside your **Hosted Zone** and applies the right **Routing Policy**.
4. It returns the best destination â€” ALB, S3 site, or CloudFront distribution.
5. The browser connects directly to that target.

### Key Advantages

* **High availability** â€” replicated servers in every region.
* **Low latency** â€” users resolved by nearest edge.
* **Health checks** â€” automatic rerouting on failure.
* **Automation** â€” full API/IaC support.
